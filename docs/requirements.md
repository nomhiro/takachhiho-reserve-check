# 要件書 — takachiho-watch

## 1. 背景

高千穂峡貸しボート予約サイト（`https://eipro.jp/takachiho1/eventCalendars/index`）で
**2026/5/8** の予約枠を確保したい。当該日の枠は人気が高く、
キャンセルによる空き発生は不定期かつ短時間で埋まると想定される。

手動の定期チェックは現実的でないため、自動監視と即時通知の仕組みを構築する。

## 2. 目的

5/8 の空き枠が発生した瞬間を捕捉し、**10分以内**にスマホへプッシュ通知を到達させる。
偽陽性（空きでないのに通知）と通知漏れを最小化する。

## 3. スコープ

### 含まれるもの
- `https://eipro.jp/takachiho1/eventCalendars/index` の 2026-05-08 の枠状況の定期取得
- 状態変化検知ロジック
- ntfy 経由の通知送信
- 監視ジョブの自己ヘルスチェック

### 含まれないもの
- 予約の自動実行（通知のみ。予約は手動で行う）
- 5/8 以外の日付の監視（将来拡張可能な設計にはしておく）
- 複数ユーザー対応

## 4. 機能要件

| ID | 要件 | 優先度 |
|---|---|---|
| FR-01 | 1時間に1回、対象ページから 5/8 の空き状況を取得する | Must |
| FR-02 | 直前の状態と比較し、状態が変化した場合のみ通知を送る | Must |
| FR-03 | 「満室 → 空きあり」検知時、urgent 優先度・専用通知音・絵文字付きで通知 | Must |
| FR-04 | 「空きあり → 満室」復帰時、default 優先度で通知 | Should |
| FR-05 | 状態を永続化し、ジョブ再実行間で保持する | Must |
| FR-06 | 毎日 7:00 JST にヘルスチェック通知を送る | Should |
| FR-07 | HTTP取得失敗・パース失敗時、リトライと連続失敗時の通知を行う | Must |
| FR-08 | 2026-05-08 以降は自動的にジョブを停止する（最終実行日は 5/7） | Should |
| FR-09 | `--dry-run` モードで通知送信を抑止し、判定結果のみログ出力できる | Must |

## 5. 非機能要件

| ID | 要件 |
|---|---|
| NFR-01 | GitHub Actions の無料枠（2,000分/月）内で完結すること。本ジョブは1日24実行 × 30秒 = 12分/月で十分余裕 |
| NFR-02 | 対象サイトへの負荷を最小化（1時間に1リクエスト、適切な User-Agent、不必要な再試行を避ける） |
| NFR-03 | GitHub Actions の cron 60日無コミット停止問題を回避（state.json の自動コミットで実コミットが定期発生するため自然に回避） |
| NFR-04 | GitHub Actions cron は最大15分の遅延が発生し得る前提で運用する |
| NFR-05 | 通知到達遅延は cron 周期 + 取得処理 + ntfy 配信で **最大75分** を許容（cron 60分 + Actions遅延15分まで） |
| NFR-06 | ntfy トピック名は推測困難な十分に長い文字列とし、Secrets で管理 |
| NFR-07 | 監視ジョブが2回連続で失敗した場合、24時間以内にユーザーが気付ける仕組みを持つ |

## 6. 制約条件

- **C-01:** 対象サイトの実装形式（SSR / CSR、内部API有無）は事前に未確認。実装フェーズ冒頭でユーザーと協力して特定する
- **C-02:** ntfy.sh の仕様上、トピック名を知る者は誰でもメッセージを送信できる。トピック名は秘匿する
- **C-03:** GitHub Actions の cron は厳密な時刻保証がない（混雑時間帯は遅延しやすい）

## 7. 成功基準

| ID | 基準 | 測定方法 |
|---|---|---|
| AC-01 | 5/8 に空きが発生してから75分以内にスマホ通知が到達する | テストトピックでの動作確認 |
| AC-02 | 偽陽性ゼロ（運用期間中に「空きあり」通知後に実際の空きが確認できないケースが無い） | 通知時にユーザーが手動でURL確認 |
| AC-03 | 監視ジョブが2回連続で失敗した場合、ヘルスチェック未着または失敗通知でユーザーが検知できる | 故意にエラーを起こして確認 |
| AC-04 | dry-run で5/8の状態を正しく判定する | ユニットテスト + 実環境dry-run |
| AC-05 | 監視期間中（着手日〜2026-05-08）に GitHub Actions の cron が継続稼働する | Actions 実行履歴の目視 |

## 8. 運用期間

- 開始: リポジトリ push 直後から
- 終了: 2026-05-08 00:00 JST（自動停止）または手動停止。最終監視日は 5/7
